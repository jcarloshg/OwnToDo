## Actor Identification

| Actor    | Type    | Description                                              |
| -------- | ------- | -------------------------------------------------------- |
| **User** | Primary | Authenticated user who creates tasks to track their work |
| **System** | Secondary | Backend that validates, saves, and provides feedback |

## Use Case Description

### UC1: Create New Task

- **Actor:** User (Primary)
- **Goal:** Create a new task with effort points to track work
- **Type:** Primary use case
- **Description:** User initiates task creation by clicking "Create Task" button, fills in required (title, effort points) and optional (description, due date) fields, and receives confirmation upon successful creation.

**Success Scenario:**
1. User clicks "Create Task" button
2. User enters task details (title, effort points, optional description, optional due date)
3. System validates input (title required, effort 1-100)
4. System persists task to database with status "pending"
5. System displays success message and updates task list

### UC2: Handle Validation Error

- **Actor:** System (Secondary)
- **Goal:** Handle and display validation failures
- **Type:** Extended (optional)
- **Description:** When validation fails, system displays an appropriate error message (e.g., "Title is required") and user remains on the form to correct errors.

**Error Scenarios:**
- Title missing or exceeds 200 characters
- Effort points not a number between 1-100
- Description exceeds 2000 characters
