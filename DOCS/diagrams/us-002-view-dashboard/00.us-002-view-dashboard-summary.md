# US-002: View Task Dashboard - Summary

## 📌 Overview

| Field          | Value                                                                                                 |
| -------------- | ----------------------------------------------------------------------------------------------------- |
| **User Story** | As a user, I want to see all my tasks on a dashboard, so that I can quickly understand my workload   |
| **Epic**       | Task Management                                                                                       |
| **Priority**   | P0 (MVP)                                                                                              |
| **Estimable**  | ~2 days (frontend + API)                                                                              |

---

## 📝 Acceptance Criteria (Gherkin)

### Scenario 1: User views dashboard with tasks

- **Given** I have tasks in the system
  - | Title           | Effort | Status    |
  - |-----------------|--------|-----------|
  - | Complete report | 5      | pending   |
  - | Review PR       | 3      | completed |
  - | Team meeting    | 2      | pending   |
- **And** I am logged into the application
- **When** I navigate to the dashboard
- **Then** I should see all 3 tasks displayed
  - And each task should show its title, effort points, and status
  - And I should see my total effort points as "10"

### Scenario 2: User filters dashboard by status

- **Given** I have tasks in the system
  - | Title           | Effort | Status    |
  - |-----------------|--------|-----------|
  - | Complete report | 5      | pending   |
  - | Review PR       | 3      | completed |
- **And** I am logged into the application
- **When** I navigate to the dashboard
  - And I filter by "pending" status
- **Then** I should only see "Complete report" task
  - And I should not see "Review PR" task

### Scenario 3: User views empty dashboard

- **Given** I have no tasks in the system
- **And** I am logged into the application
- **When** I navigate to the dashboard
- **Then** I should see an empty state message "No tasks yet. Create your first task!"
  - And I should see a "Create Task" call-to-action button

## 🎯 Use Cases Breakdown

| Use Case                     | Type        | Description                                                              |
| ---------------------------- | ----------- | ------------------------------------------------------------------------ |
| **UC1: View Dashboard**      | Primary     | User navigates to dashboard and views task list                          |
| **UC2: Display Task List**    | <<include>> | Render all tasks with title, effort points, status                       |
| **UC3: Calculate Totals**     | <<include>> | Sum effort points for display                                            |
| **UC4: Filter by Status**     | <<include>> | Allow user to filter tasks by pending/completed                          |
| **UC5: Show Empty State**     | <<extend>>  | When no tasks exist, display empty state with CTA                        |
| **UC6: Navigate to Create**   | <<extend>>  | User can click CTA to navigate to task creation                          |
