## Use Case Description

### UC1: View Dashboard

- **Actor:** User
- **Goal:** View all tasks to understand workload
- **Type:** Primary use case
- **Description:** User navigates to the dashboard and sees an overview of all their tasks with summary information.

### UC2: Display Task List

- **Actor:** User (via system)
- **Goal:** Show all tasks with details
- **Type:** Included (required)
- **Description:** System renders the list of tasks showing title, effort points, and status for each task.

### UC3: Calculate Totals

- **Actor:** System
- **Goal:** Show aggregate information
- **Type:** Included (required)
- **Description:** System sums up effort points from all visible tasks to display total workload.

### UC4: Filter by Status

- **Actor:** User (via system)
- **Goal:** View subset of tasks
- **Type:** Included (required)
- **Description:** User can filter tasks by status (all/pending/completed) to focus on specific items.

### UC5: Show Empty State

- **Actor:** System
- **Goal:** Guide user when no tasks exist
- **Type:** Extended (optional)
- **Description:** When user has no tasks, display friendly message with call-to-action.

### UC6: Navigate to Create

- **Actor:** System
- **Goal:** Enable task creation
- **Type:** Included (from empty state)
- **Description:** User can click CTA button to navigate to task creation screen.
