## Actor Identification

| Actor    | Type    | Description                                              |
| -------- | ------- | -------------------------------------------------------- |
| **User** | Primary | Authenticated user who creates tasks to track their work |

## Use Case Description

### UC1: Create New Task

- **Actor:** User
- **Goal:** Create a new task with effort points to track work
- **Type:** Primary use case
- **Description:** User initiates task creation by clicking "Create Task" button and fills in required and optional fields.

### UC2: Enter Task Details

- **Actor:** User (via system)
- **Goal:** Capture task information
- **Type:** Included (required)
- **Description:** User enters title, description (optional), effort points, and due date (optional).

### UC3: Validate Input

- **Actor:** System
- **Goal:** Ensure data integrity
- **Type:** Included (required)
- **Description:** System validates:
  - Title is required (1-200 characters)
  - Effort points must be integer 1-100
  - Description max 2000 characters (optional)

### UC4: Save Task

- **Actor:** System (Database)
- **Goal:** Persist task to database
- **Type:** Included (required)
- **Description:** System saves task with status "pending" and timestamps.

### UC5: Show Success Message

- **Actor:** System
- **Goal:** Confirm action completed
- **Type:** Included (required)
- **Description:** Display "Task created successfully" toast and update task list.

### UC6: Show Error Message

- **Actor:** System
- **Goal:** Handle validation failures
- **Type:** Extended (optional)
- **Description:** Display appropriate error message when validation fails (e.g., "Title is required").
