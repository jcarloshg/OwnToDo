# SLOs and SLIs - TaskFlow

## Overview

This document defines the **Service Level Objectives (SLOs)** and **Service Level Indicators (SLIs)** for TaskFlow, ensuring the application meets performance, availability, and reliability standards.

---

## 1. Availability SLOs

| SLO                     | Target          | Description                                                 |
| ----------------------- | --------------- | ----------------------------------------------------------- |
| **Uptime**              | 99.5%           | Application should be available 99.5% of the time per month |
| **Planned Maintenance** | ≤ 4 hours/month | Scheduled maintenance windows should not exceed 4 hours     |

### SLIs for Availability

| Indicator               | Measurement                                            |
| ----------------------- | ------------------------------------------------------ |
| **Successful Requests** | `(Successful requests / Total requests) × 100`         |
| **Error Rate**          | `(HTTP 5xx responses / Total requests) × 100`          |
| **Downtime**            | Total minutes where `/health` endpoint returns non-200 |

**Error Budget Policy:**

- If error rate exceeds 0.5% in a 24-hour window → Alert on-call
- If uptime drops below 99.5% in a month → Post-mortem required

## 2. Performance SLOs

| SLO                   | Target        | Measurement Window |
| --------------------- | ------------- | ------------------ |
| **Page Load Time**    | < 2s (p95)    | Continuous         |
| **API Response Time** | < 300ms (p95) | Continuous         |
| **Graph Render Time** | < 1s (p95)    | Continuous         |

### SLIs for Performance

| Indicator             | Measurement                                         |
| --------------------- | --------------------------------------------------- |
| **Page Load (p95)**   | 95th percentile of `domContentLoaded` event         |
| **API Latency (p95)** | 95th percentile of `/api/*` endpoint response times |
| **Graph Render**      | Time from data received to chart displayed          |

**Targets by Endpoint:**

| Endpoint               | Target Latency (p95) |
| ---------------------- | -------------------- |
| `GET /api/tasks`       | 150ms                |
| `POST /api/tasks`      | 200ms                |
| `GET /api/analytics/*` | 300ms                |
| `POST /api/auth/login` | 250ms                |

---

## 3. Reliability SLOs

| SLO                       | Target | Description                             |
| ------------------------- | ------ | --------------------------------------- |
| **Task Creation Success** | ≥ 99%  | Tasks should save successfully          |
| **Data Durability**       | 99.99% | No data loss for committed transactions |

### SLIs for Reliability

| Indicator                      | Measurement                                               |
| ------------------------------ | --------------------------------------------------------- |
| **Creation Success Rate**      | `(Successful task creates / Total create attempts) × 100` |
| **Transaction Rollback Rate**  | `(Rollbacks / Total transactions) × 100`                  |
| **Database Connection Errors** | Count of connection failures per hour                     |

---

## 4. Scalability SLOs

| SLO                  | Target         | Description                              |
| -------------------- | -------------- | ---------------------------------------- |
| **Concurrent Users** | ≥ 100 users    | Support 100 simultaneous active sessions |
| **API Throughput**   | ≥ 1000 req/min | Handle 1000 API requests per minute      |

### SLIs for Scalability

| Indicator               | Measurement                                         |
| ----------------------- | --------------------------------------------------- |
| **Active Sessions**     | Count of concurrent authenticated sessions          |
| **Request Queue Depth** | Number of waiting requests when load > 80% capacity |
| **CPU Utilization**     | Average CPU usage should stay below 70%             |

## 6. Security SLOs

| SLO                       | Target                          | Description                                    |
| ------------------------- | ------------------------------- | ---------------------------------------------- |
| **Failed Login Block**    | After 5 attempts                | Lock account after 5 consecutive failed logins |
| **Session Timeout**       | 30 minutes                      | Auto-logout after 30 minutes of inactivity     |
| **Password Requirements** | 8+ chars, 1 uppercase, 1 number | Enforce strong password policy                 |

## 7. Error Budget

| Category                | Budget                  | Reset Period |
| ----------------------- | ----------------------- | ------------ |
| Downtime                | 3.6 hours/month         | Monthly      |
| API Errors (5xx)        | 0.5% of requests        | Monthly      |
| Performance Degradation | 5% of requests over SLA | Monthly      |

---

## 8. Monitoring & Alerting

### Key Dashboards

1. **Overview Dashboard**
   - Current uptime
   - Error rate (real-time)
   - Active users
   - Response time trends

2. **Performance Dashboard**
   - API latency percentiles (p50, p95, p99)
   - Page load times
   - Database query performance

3. **Business Dashboard**
   - Daily active users
   - Tasks created/completed
   - Completion rate trends

### Alerting Rules

| Alert                   | Condition              | Severity  |
| ----------------------- | ---------------------- | --------- |
| **High Error Rate**     | 5xx > 1% for 5 min     | PagerDuty |
| **Slow API**            | p95 > 500ms for 10 min | Slack     |
| **Database Connection** | > 10 failures/hour     | PagerDuty |
| **Uptime Breach**       | < 99.5% in month       | Email     |
