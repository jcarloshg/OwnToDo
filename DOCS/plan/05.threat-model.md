# Threat Model - TaskFlow

## Overview

This threat model identifies security threats for TaskFlow using the **STRIDE** methodology (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege).

## 2. STRIDE Threat Analysis

### S - Spoofing (Authentication Threats)

| Threat | Description | Attack Vector | Mitigation |
|--------|-------------|---------------|------------|
| **T-S-01** | User impersonation via stolen credentials | Phishing, credential stuffing | Strong password policy, rate limiting on login |
| **T-S-02** | Session hijacking | XSS窃取JWT, MITM | HTTP-only cookies, short session expiry (30 min) |
| **T-S-03** | Brute force login | Automated password guessing | 5 max attempts, lock account, CAPTCHA |

### T - Tampering (Integrity Threats)

| Threat | Description | Attack Vector | Mitigation |
|--------|-------------|---------------|------------|
| **T-T-01** | Task data modification | Unauthorized API calls | User ownership validation on all endpoints |
| **T-T-02** | Database injection | Malicious SQL via input | Parameterized queries, input validation |
| **T-T-03** | Graph data manipulation | Modifying API responses | Validate all data client-side, server-side checks |

### R - Repudiation (Non-Repudiation)

| Threat | Description | Attack Vector | Mitigation |
|--------|-------------|---------------|------------|
| **T-R-01** | User denies action | No audit log | Log all state-changing operations with timestamp |
| **T-R-02** | Admin denies configuration change | No audit trail | Immutable audit log |

### I - Information Disclosure (Confidentiality)

| Threat | Description | Attack Vector | Mitigation |
|--------|-------------|---------------|------------|
| **T-I-01** | Database exposure | SQL injection, backup theft | Encrypt sensitive data at rest, HTTPS in transit |
| **T-I-02** | Credential disclosure | Leaked env files | Never commit .env, use secrets manager |
| **T-I-03** | Task data exposure | Unauthorized user access | Strict ownership checks on all queries |

### D - Denial of Service

| Threat | Description | Attack Vector | Mitigation |
|--------|-------------|---------------|------------|
| **T-D-01** | API flooding | High request volume | Rate limiting (100 req/min per user) |
| **T-D-02** | Database exhaustion | Large result sets | Pagination (max 100 items/page) |
| **T-D-03** | Memory exhaustion | Complex graph queries | Query timeouts, data aggregation limits |

### E - Elevation of Privilege

| Threat | Description | Attack Vector | Mitigation |
|--------|-------------|---------------|------------|
| **T-E-01** | Unauthorized admin access | Direct API manipulation | No admin role in MVP, single user model |
| **T-E-02** | Bypass authentication | Token manipulation | JWT signature validation, expiration check |

## 4. Security Controls

### Authentication
| Control | Implementation |
|---------|---------------|
| Password Hashing | bcrypt (cost factor 12) |
| Session Token | JWT with 30-min expiration |
| Password Policy | Min 8 chars, 1 uppercase, 1 number |

### Authorization
| Control | Implementation |
|---------|---------------|
| Task Ownership | All queries filtered by `user_id` from JWT |
| API Validation | Middleware validates JWT on every protected route |

### Input Validation
| Input | Validation |
|-------|------------|
| Task Title | Max 200 chars, no HTML/script tags |
| Effort Points | Integer 1-100 |
| Description | Max 2000 chars, sanitized |

### Data Protection
| Data | Protection |
|------|------------|
| Passwords | bcrypt hashed |
| JWT Secret | Environment variable, never committed |
| Database | Local file storage in MVP |

## 6. Security Incident Response

| Severity | Definition | Response |
|----------|------------|----------|
| **Critical** | Data breach, credential compromise | Immediate alert, all sessions revoked |
| **High** | Unauthorized access attempt, injection | Log and alert, investigate |
| **Medium** | Rate limit exceeded, suspicious activity | Log, temporary block |
| **Low** | Invalid input attempts | Log, reject request |
