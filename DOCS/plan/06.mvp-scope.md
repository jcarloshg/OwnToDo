# MVP Scope - TaskFlow

## Overview

This document defines the **Minimum Viable Product (MVP)** scope for TaskFlow, separating **Must-Have** features from **Nice-to-Have** features to ensure focused development.

---

## MVP Goals

| Goal | Target |
|------|--------|
| **Time to Market** | 2-3 weeks |
| **Core Value** | Task tracking with effort points and basic analytics |
| **Tech Stack** | React + Express.js + PostgreSQL |

---

## Must-Have (MVP Scope)

### Authentication
| Feature | Description | Priority |
|---------|-------------|----------|
| User Registration | Email + password signup | P0 |
| User Login | Secure login with JWT | P0 |
| Session Management | 30-minute timeout | P0 |

### Task Management
| Feature | Description | Priority |
|---------|-------------|----------|
| Create Task | Title, description (optional), effort points (1-10) | P0 |
| View Dashboard | List all tasks with status | P0 |
| Complete Task | Mark task as done | P0 |
| Delete Task | Remove task permanently | P1 |
| Edit Task | Update task details | P1 |

### Analytics
| Feature | Description | Priority |
|---------|-------------|----------|
| Daily Graph | Bar chart showing completed vs pending effort | P1 |
| Task Count | Total tasks, completed count | P0 |

### Infrastructure
| Feature | Description | Priority |
|---------|-------------|----------|
| Health Check | `/health` endpoint | P0 |
| Error Handling | Graceful error responses | P0 |
| Input Validation | Sanitize all user inputs | P0 |

---

## Out of Scope (Not Planned)

| Feature | Reason |
|---------|--------|
| Mobile App | React Native would be separate project |
| Calendar Integration | Google Calendar, Outlook sync |
| Notifications | Email/push notifications |
| Offline Mode | Requires significant additional work |
| API Public Access | Internal use only |
| Billing/Subscription | Personal use, no monetization |

---

## Technical Scope Boundaries

### In Scope
- Single-user model (no multi-tenancy)
- Local PostgreSQL database
- RESTful API design
- React with functional components and hooks
- Docker Compose for local development

### Architecture Decisions

```
┌─────────────────────────────────────────────────────────────┐
│                        MVP Architecture                      │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   Frontend          API Layer           Data Layer           │
│   ┌────────┐       ┌──────────┐        ┌───────────┐       │
│   │ React  │──────►│ Express  │───────►│ PostgreSQL│       │
│   │ SPA    │◄──────│ REST API │◄───────│           │       │
│   └────────┘       └──────────┘        └───────────┘       │
│                                                              │
│   State: React Query / Context                              │
│   Auth: JWT in HTTP-only cookie                            │
│   Charts: Recharts                                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## Definition of Done

### For a Feature to be "Done"
- [ ] Code implemented and reviewed
- [ ] Unit tests written and passing (≥ 80% coverage)
- [ ] Integration tests passing
- [ ] No critical/high security findings
- [ ] Runs locally with Docker Compose
- [ ] Acceptance criteria met (per Gherkin scenarios)
