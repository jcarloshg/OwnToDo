# User Stories - TaskFlow

## Epic 1: Task Management

---

### US-001: Create a New Task

**As a** user, **I want to** create a new task with effort points, **so that** I can track the work needed for my activities.

**Acceptance Criteria (Gherkin):**

```gherkin
Feature: Create New Task

  Scenario: User creates a task with all fields
    Given I am logged into the application
    And I am on the dashboard
    When I click the "Create Task" button
    And I enter "Complete project report" as the task title
    And I enter "Write the final Q1 report for the marketing team" as the description
    And I enter "5" as the effort points
    And I select "2026-04-25" as the due date
    And I click the "Save" button
    Then I should see a success message "Task created successfully"
    And the task "Complete project report" should appear in my task list
    And the task should have 5 effort points

  Scenario: User creates a task with minimal fields
    Given I am logged into the application
    And I am on the dashboard
    When I click the "Create Task" button
    And I enter "Buy groceries" as the task title
    And I enter "2" as the effort points
    And I click the "Save" button
    Then I should see a success message "Task created successfully"
    And the task "Buy groceries" should appear in my task list

  Scenario: User tries to create task without required fields
    Given I am logged into the application
    And I am on the dashboard
    When I click the "Create Task" button
    And I click the "Save" button
    Then I should see an error message "Title is required"
    And the form should remain open
```

**INVEST Checklist:**
- ✅ Independent: Can be implemented independently
- ✅ Negotiable: Effort points and due date are optional
- ✅ Valuable: Core functionality for task management
- ✅ Estimable: ~3 days for frontend + API
- ✅ Small: Single focused feature
- ✅ Testable: Clear acceptance criteria

### US-002: View Task Dashboard

**As a** user, **I want to** see all my tasks on a dashboard, **so that** I can quickly understand my workload.

**Acceptance Criteria (Gherkin):**

```gherkin
Feature: View Task Dashboard

  Scenario: User views dashboard with tasks
    Given I have tasks in the system
      | Title              | Effort | Status    |
      | Complete report    | 5      | pending   |
      | Review PR          | 3      | completed |
      | Team meeting       | 2      | pending   |
    And I am logged into the application
    When I navigate to the dashboard
    Then I should see all 3 tasks displayed
    And each task should show its title, effort points, and status
    And I should see my total effort points as "10"

  Scenario: User filters dashboard by status
    Given I have tasks in the system
      | Title              | Effort | Status    |
      | Complete report    | 5      | pending   |
      | Review PR          | 3      | completed |
    And I am logged into the application
    When I navigate to the dashboard
    And I filter by "pending" status
    Then I should only see "Complete report" task
    And I should not see "Review PR" task

  Scenario: User views empty dashboard
    Given I have no tasks in the system
    And I am logged into the application
    When I navigate to the dashboard
    Then I should see an empty state message "No tasks yet. Create your first task!"
    And I should see a "Create Task" call-to-action button
```

**INVEST Checklist:**
- ✅ Independent: Dashboard can be built separately
- ✅ Negotiable: Filtering options can be adjusted
- ✅ Valuable: Core visibility requirement
- ✅ Estimable: ~2 days
- ✅ Small: Focused on display
- ✅ Testable: Clear expected outputs

### US-003: Complete a Task

**As a** user, **I want to** mark a task as complete, **so that** I can track my progress.

**Acceptance Criteria (Gherkin):**

```gherkin
Feature: Complete Task

  Scenario: User completes a pending task
    Given I have a task "Write documentation" with effort 3 and status "pending"
    And I am logged into the application
    And I am on the dashboard
    When I click the "Complete" button on "Write documentation" task
    Then the task status should change to "completed"
    And the "Complete" button should change to "Undo" button
    And I should see a toast message "Task completed!"

  Scenario: User undoes a completed task
    Given I have a task "Write documentation" with effort 3 and status "completed"
    And I am logged into the application
    And I am on the dashboard
    When I click the "Undo" button on "Write documentation" task
    Then the task status should change to "pending"
    And the "Undo" button should change to "Complete" button
```

**INVEST Checklist:**
- ✅ Independent: Can be implemented independently
- ✅ Negotiable: Undo feature is optional
- ✅ Valuable: Core tracking functionality
- ✅ Estimable: ~1 day
- ✅ Small: Single action
- ✅ Testable: Clear state changes

## Epic 2: Analytics & Visualization

---

### US-006: View Daily Task Graph

**As a** user, **I want to** see a graph of my tasks for today, **so that** I can track my daily productivity.

**Acceptance Criteria (Gherkin):**

```gherkin
Feature: Daily Task Graph

  Scenario: User views daily completion graph
    Given I have completed 5 tasks today with a total of 15 effort points
    And I have 3 pending tasks with a total of 8 effort points
    And I am logged into the application
    When I navigate to the "Analytics" section
    And I select "Daily" view
    Then I should see a bar chart showing completed (15) vs pending (8) effort points
    And the chart should display today's date as the title
```

---

```gherkin
Feature: Weekly Task Graph

```

## Epic 3: User Management

---

### US-010: User Registration

**As a** new user, **I want to** register for an account, **so that** I can start tracking my tasks.

**Acceptance Criteria (Gherkin):**

```gherkin
Feature: User Registration

  Scenario: New user registers successfully
    Given I am on the registration page
    When I enter "alex@example.com" as email
    And I enter "SecurePass123!" as password
    And I enter "SecurePass123!" as confirm password
    And I click "Sign Up"
    Then I should see a success message "Account created!"
    And I should be automatically logged in
    And I should be redirected to the dashboard
```

---

### US-011: User Login

**As a** registered user, **I want to** log into the application, **so that** I can access my tasks.

**Acceptance Criteria (Gherkin):**

```gherkin
Feature: User Login

  Scenario: User logs in successfully
    Given I have an account with email "alex@example.com" and password "SecurePass123!"
    And I am on the login page
    When I enter "alex@example.com" as email
    And I enter "SecurePass123!" as password
    And I click "Log In"
    Then I should see a success message "Welcome back!"
    And I should be redirected to the dashboard

  Scenario: User fails login with wrong password
    Given I have an account with email "alex@example.com" and password "SecurePass123!"
    And I am on the login page
    When I enter "alex@example.com" as email
    And I enter "WrongPassword" as password
    And I click "Log In"
    Then I should see an error message "Invalid email or password"
```
